# v0.29.0 Runtime Foundation Correction Plan

## Purpose

`v0.29.0` was the active version when this correction was executed. The plan did
not add a new roadmap version; it repaired the runtime foundation before that
release branch was merged so later conversation, hook, skill, TUI, subagent,
team, and plugin work could reuse one authority model instead of extending a
patch-only state machine. v0.29.1 retains this correction and changes only the
release checksum pipeline.

Status: C1-C7 complete. The correction implementation, documentation alignment,
independent reviews, and final local release-gate verification are complete.

## Product Invariants

- `run` supports read-only answers, inspection, planning, and patch proposals.
- The runtime core owns context, policy, state, approvals, evidence, and final
  reporting. CLI surfaces only parse input and render runtime results.
- Ontology lookup is the first context-selection layer. Authoritative source is
  reread before an edit proposal is accepted.
- A workflow is a generic envelope. Patch details are typed action data, not the
  workflow identity itself.
- Patch application approval never grants verification-command execution.
- Corrupt canonical state fails closed. SQLite remains a rebuildable projection.
- Concurrent ledger writers serialize without losing or forking events.
- Raw model output is not a user-facing final or approval report.
- Tests distinguish deterministic fixtures from measurements made with an
  actual Qwen or Gemma candidate.

## Non-Goals

- Promoting a default Qwen or Gemma model; that remains `v0.30.0` and requires
  source-backed local evidence.
- Streaming and cancellation of backend generation (`v0.31.0`).
- Full transcript replay (`v0.32.0`).
- Executable hooks, skills, TUI actions, subagents, teams, or plugin adapters
  (`v0.33.0` through `v0.38.0`).
- Replacing the canonical ontology store with RDF/OWL solely for this release.

## Release Gate

`v0.29.0` must not merge into `main` until C1-C7 are complete, targeted tests
pass, the full locked test suite passes serially, and an independent reviewer
confirms that the correction did not turn deferred roadmap items into claimed
current capabilities.
