# v0.48.0 웹 연구와 제한된 브라우저 계획

## 목적

`v0.48.0`은 현재의 단발성 `WebSearch`, `WebOpen`, `WebFind`를 작은 로컬
모델에서도 예측 가능하게 작동하는 제한된 웹 연구 흐름으로 확장합니다. 검색 결과
snippet만으로 답하지 않고, 필요한 결과를 열고 문서 안에서 근거를 찾은 뒤 검증된
출처를 답변에 연결합니다.

동적 웹페이지나 실제 사이트 조작이 필요한 요청에는 설치된 Chromium 계열
브라우저를 격리된 임시 profile로 실행하는 제한된 browser action을 제공합니다.
예를 들어 사용자가 “네이버를 열고 검색창에 월드컵을 입력해”라고 요청하면
`navigate -> observe -> type -> press -> extract` 순서로 실제 네이버 페이지를
조작합니다.

이 문서는 구현 계획입니다. 현재 `feature/tui-interactions-search` branch와 PR에는
이 기능을 구현하지 않습니다.

## 원칙과 경계

- 검색, URL 읽기, 페이지 찾기와 browser action은 저장소가 소유하는 typed
  interface로 구현합니다.
- 검색 provider API, API key, MCP server, provider SDK, Node.js, Puppeteer,
  별도 background 검색 service를 필수 구성으로 추가하지 않습니다.
- 기존의 API key 없는 직접 검색을 유지하고 DuckDuckGo HTML과 lite 문서를
  서로 보완하는 fallback으로 사용합니다.
- 모델은 임의 URL, shell, JavaScript 또는 browser process 권한을 직접 얻지
  않습니다. Runtime이 검증된 typed action만 실행합니다.
- 모든 웹 자료와 DOM text는 신뢰하지 않는 근거입니다. 웹 문서의 지시는 tool
  권한이나 쓰기 권한으로 승격되지 않습니다.
- 4096-token 모델에서도 동작하도록 tool 횟수, network 요청, 근거 크기와 최종
  생성 token을 각각 제한합니다.
- 기존 사용자 browser profile, cookie, login session, 저장된 password와
  extension을 사용하지 않습니다.
- browser가 없거나 시작하지 못해도 정적 `WebSearch`, `WebOpen`, `WebFind`와
  일반 로컬 대화는 계속 사용할 수 있어야 합니다.

## 목표 동작

### 제한된 웹 연구

```text
사용자 질문
  -> 검색 필요성 판정
  -> 검색어 생성 또는 결정적 fallback
  -> WebSearch
  -> 결과 품질·중복 평가
  -> 최대 3개 결과 WebOpen
  -> 필요한 문서 WebFind
  -> 근거별 source id 검증
  -> 출처가 연결된 최종 답변
```

한 요청의 기본 상한은 다음과 같습니다.

| 항목 | 상한 |
| --- | --- |
| 전체 web tool step | 6 |
| WebSearch | 2 |
| WebOpen | 3 |
| 검색어 수정 | 1 |
| WebFind | 열린 문서당 2 |
| 외부 network 요청 | 6 |
| 최종 모델용 web evidence | 8 KiB |
| 전체 web elapsed time | 45초 |
| 최종 답변 생성 | 768 tokens |

모델이 tool action을 올바르게 만들지 못하면 현재의 명시적 명령과
freshness keyword routing을 결정적 fallback으로 사용합니다. 같은 입력으로 같은
실패를 두 번 넘게 재시도하지 않습니다.

### 직접 검색과 문서 읽기

- `WebSearch`는 DuckDuckGo HTML 요청이 제한되거나 parser contract와 맞지 않으면
  DuckDuckGo lite 문서를 한 번 시도합니다.
- 검색 결과는 canonical HTTPS URL로 중복 제거하고 동일 domain 편중을 제한합니다.
- 공식 문서, 원문, release note와 사용자 요청에 직접 관련된 결과를 우선합니다.
- `WebOpen`은 `main`, `article`과 text-density를 이용해 본문을 우선 추출하고
  `nav`, `aside`, `footer`, script, style과 반복 boilerplate를 제외합니다.
- HTML, plain text, JSON에 더해 Markdown, RSS와 Atom text를 지원합니다.
- PDF와 Office 문서는 이 버전의 필수 범위가 아닙니다. 지원할 경우 별도의 크기,
  dependency, parser 보안 검토를 통과해야 합니다.
- `WebFind`는 literal match를 기본으로 유지하면서 앞뒤 문맥과 source id를
  반환합니다. regex나 임의 code 실행은 지원하지 않습니다.

### 근거와 인용

- 검색 결과와 열린 문서마다 runtime이 stable `source_id`를 발급합니다.
- 모델은 runtime이 제공한 `source_id`만 인용할 수 있습니다.
- Runtime은 존재하지 않는 인용, URL 변조와 source 간 바꿔치기를 제거합니다.
- 최종 TUI는 단순 URL 모음 대신 주장 또는 문단과 연결된 검증된 출처를 표시합니다.
- 근거가 충돌하거나 최신성을 판단할 수 없으면 답변에 불확실성을 표시합니다.
- 모델 요약이 실패해도 열린 문서의 제목, 검증된 URL과 제한된 발췌는 보존합니다.

### 여러 페이지와 TUI

- 단일 `opened_web_page` 대신 최대 8개의 bounded page session을 유지합니다.
- `/sources`는 검색 결과와 열린 페이지를 keyboard picker로 표시합니다.
- 사용자는 방향키와 Enter로 source를 열고 `/find` 대상으로 선택할 수 있습니다.
- TUI는 `검색 -> 결과 평가 -> 문서 읽기 -> 근거 구성 -> 답변 생성` 상태를
  사용자 용어로 표시합니다.
- `backend`, `workflow id`, raw tool payload 같은 내부 용어는 일반 대화 화면에
  노출하지 않습니다.

## 제한된 Chromium browser

### 실행 경계

- macOS, Linux와 Windows에 이미 설치된 Chrome, Chromium, Edge 계열 executable을
  탐색합니다. 별도의 Chromium bundle을 자동 다운로드하지 않습니다.
- 매 요청은 관리형 임시 user-data directory와 빈 profile을 사용합니다.
- Runtime이 browser process와 DevTools 연결의 생성, timeout, 종료와 잔여 process
  정리를 소유합니다.
- 공개 HTTPS URL만 탐색하며 `localhost`, private/link-local IP, credential URL,
  `file:`, `data:`, `javascript:` scheme을 차단합니다.
- 기존 사용자 browser의 CDP endpoint나 login session에는 자동 연결하지 않습니다.
- CAPTCHA, login, 결제, 게시, 댓글, upload, download, permission prompt와
  browser extension 조작은 수행하지 않고 명시적인 차단 결과를 반환합니다.

### 허용 action

| Action | 의미 |
| --- | --- |
| `navigate` | 검증된 공개 HTTPS 페이지 열기 |
| `observe` | 접근성 role, name과 runtime 발급 element handle 확인 |
| `click` | 확인된 handle의 읽기 전용 navigation 또는 control 선택 |
| `type` | 확인된 text field에 bounded text 입력 |
| `press` | Enter, Escape, 방향키 등 닫힌 key 목록 전송 |
| `scroll` | bounded viewport 이동 |
| `extract` | 현재 페이지의 제한된 readable text 추출 |
| `screenshot` | 현재 viewport의 제한된 local artifact 생성 |
| `close` | tab과 임시 browser session 종료 |

모델이 만든 CSS selector, XPath, JavaScript와 Node code는 실행하지 않습니다.
`observe`가 발급한 opaque element handle만 후속 action에서 사용할 수 있으며,
navigation이나 DOM revision이 바뀌면 기존 handle은 만료됩니다.

### 대표 시나리오

“네이버를 열고 검색창에 월드컵을 입력해” 요청은 다음 계약으로 실행합니다.

1. `navigate("https://www.naver.com/")`
2. `observe`로 검색 text field와 submit control 확인
3. 검색 field handle에 `type("월드컵")`
4. `press("ENTER")` 또는 확인된 submit handle `click`
5. 결과 페이지를 다시 `observe`하고 `extract`
6. 검증된 결과 URL과 text를 제한된 근거로 최종 답변에 전달
7. browser tab, process와 임시 profile 정리

Google, Naver처럼 특정 사이트 이름을 hard-code한 selector를 두지 않습니다.
동일한 접근성 기반 action 계약을 사용하며 사이트 구조가 달라져 안전하게 대상을
확인할 수 없으면 추측해서 click하지 않습니다.

## 예상 ownership

| 영역 | 현재 owner와 예상 변경 |
| --- | --- |
| web transport·SSRF policy | `src/adapters/web_search/transport.rs`, `policy.rs`의 실제 resolver 검사를 유지 |
| 검색 parsing·evidence | `src/adapters/web_search/html.rs`, `evidence.rs`에 dual parser와 source id 추가 |
| 본문 추출·find | `src/adapters/web_search/page.rs`, `find.rs`에 main-content와 contextual match 추가 |
| 연구 orchestration | `src/app/web_search_adapter.rs` 아래에 bounded step owner 추가 |
| TUI page session·source picker | `src/app/tui_adapter`, `src/surfaces/tui`가 사용자 상호작용만 소유 |
| browser process·CDP | 새 `src/adapters/browser` 경계가 executable, session, protocol과 network policy 소유 |
| browser action policy | 새 runtime-core typed request/result가 허용 action과 budget을 소유 |
| transcript·compaction | 기존 sanitized tool/evidence record와 context budget을 재사용 |

브라우저 구현에 새 Rust crate가 필요하면 구현 전에 license, binary size, Windows
linkage, maintenance와 protocol surface를 문서화합니다. Node.js runtime이나
외부 browser automation process를 도입하는 선택은 이 계획의 기본안이 아닙니다.

## 구현 순서와 커밋 경계

| 단위 | 작업 | targeted 검증 | 커밋 경계 |
| --- | --- | --- | --- |
| W1 | typed web step, budget과 결정적 fallback을 추가 | routing·budget·loop terminal unit test | `feat(web): add bounded research orchestration` |
| W2 | DDG HTML/lite fallback, ranking과 canonical source id 추가 | dual fixture·dedup·parser drift test | `feat(search): harden direct result discovery` |
| W3 | main-content extraction, RSS/Atom/Markdown과 contextual find 추가 | hostile HTML·format·budget fixture | `feat(web): improve readable page evidence` |
| W4 | claim/source 검증과 multi-page session 추가 | invalid citation·page eviction·resume test | `feat(web): bind answers to verified sources` |
| W5 | `/sources` picker와 web progress rendering 추가 | native terminal picker·narrow-width snapshot | `feat(tui): add web evidence navigation` |
| B1 | installed browser discovery, 격리 process와 제한된 protocol client 추가 | fake browser·process cleanup·platform test | `feat(browser): add isolated chromium session` |
| B2 | observe와 opaque handle 기반 action 추가 | stale handle·forbidden action·budget test | `feat(browser): add bounded page interaction` |
| B3 | Naver/Google 일반 검색 form E2E와 TUI 상태 추가 | deterministic fake page와 opt-in live smoke | `feat(tui): integrate browser research actions` |
| V1 | 문서, threat model, privacy와 candidate preflight 정렬 | link·marker·targeted·PR candidate gate | `docs(web): document research and browser boundaries` |

각 단위는 `구현 -> targeted 검증 -> 커밋 -> push`로 닫습니다. 기능 단위 개발 중
전체 test와 release gate를 반복하지 않으며, 최종 candidate HEAD에서만 PR CI가
전체 검증을 수행합니다.

## 수용 기준

- 일반 최신성 질문이 최대 budget 안에서 검색 결과를 찾고, 상위 문서를 열고,
  검증된 source id가 연결된 한국어 답변을 반환합니다.
- 검색 결과 snippet과 열린 본문이 충돌하면 snippet만으로 단정하지 않습니다.
- DDG HTML fixture가 변경되거나 결과가 없을 때 lite fallback 또는 명확한
  bounded failure로 종료합니다.
- `/sources`에서 최대 8개 source를 방향키로 선택하고 열거나 현재 `/find`
  대상으로 지정할 수 있습니다.
- “네이버를 열고 검색창에 월드컵을 입력해” deterministic browser E2E가
  `navigate`, `observe`, `type`, `press`, `extract`, cleanup 순서를 증명합니다.
- browser action은 raw JavaScript, arbitrary selector, local/private URL, 기존
  user profile, upload/download와 인증된 side effect를 실행할 수 없습니다.
- browser executable이 없는 환경에서는 설치 안내를 표시하고 기존 정적 web
  tool과 일반 대화가 정상 동작합니다.
- network, model과 browser fixture는 기본 test에서 결정적으로 대체되며 live
  smoke는 명시적 opt-in에서만 실행됩니다.
- macOS ARM64/Intel, Linux x86_64/ARM64와 Windows x86_64 candidate가 같은
  typed contract로 compile되고 platform-specific browser absence를 안전하게
  처리합니다.

## 검증

### Unit과 fixture

- 검색 query, step count, network count, elapsed와 evidence budget
- DDG HTML/lite 정상·빈 결과·anti-bot·malformed fixture
- URL canonicalization, DNS rebinding, redirect, private IP와 deceptive authority
- main-content, 반복 boilerplate, hostile tag, RSS/Atom/Markdown
- source id 생성, invalid citation 제거와 conflicting evidence
- browser executable 탐색, protocol frame 상한과 timeout
- stale element handle, popup, download, upload, login과 금지 scheme 차단

### Integration과 TUI

- search-open-find-answer 전체 흐름을 network 없는 fake transport로 검증
- 여러 source 선택, page eviction, `/sources`, `/find`와 context compaction
- browser fake page에서 검색 form 관찰·입력·제출·추출·cleanup 검증
- narrow terminal, wide character, 취소와 browser process 종료 검증

### Candidate gate

- 변경 모듈 targeted test
- `cargo fmt --all -- --check`
- 최종 PR candidate HEAD의 전체 locked test, clippy, build와 Windows compile
- 명시적 opt-in live DDG WebSearch/WebOpen smoke
- 지원 platform에서 명시적 opt-in Chromium smoke

## 비목표

- Google, Naver 또는 상용 provider의 비공개 API 우회
- CAPTCHA 우회, stealth fingerprint spoofing과 anti-bot 회피
- 사용자 browser profile, cookie와 로그인 상태 재사용
- 임의 JavaScript, Node.js, shell 또는 filesystem browser script
- browser를 이용한 게시, 결제, 계정 변경과 파일 전송
- Chromium binary 자동 다운로드나 별도 browser 배포 channel 운영
- 모든 웹사이트의 완전한 rendering 호환성 보장
