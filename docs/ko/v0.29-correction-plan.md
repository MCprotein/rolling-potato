# v0.29.0 런타임 기반 보정 계획

## 목적

활성 버전은 `v0.29.0`으로 유지합니다. 새 로드맵 버전을 추가하지 않고
release branch를 merge하기 전에 런타임 기반을 보정합니다. 이후 대화,
hook, skill, TUI, subagent, team, plugin이 patch 전용 상태기계를 확장하지
않고 하나의 권위 모델을 재사용할 수 있어야 합니다.

## 제품 불변식

- `run`은 읽기 전용 답변, 저장소 조회, 계획, patch 제안을 모두 처리한다.
- runtime core가 context, policy, state, approval, evidence, 최종 보고를
  소유한다. CLI surface는 입력 파싱과 runtime 결과 표시만 담당한다.
- 온톨로지 조회가 첫 번째 context 선택 계층이다. 수정안을 받아들이기
  전에는 authoritative source를 다시 읽는다.
- workflow는 범용 envelope다. Patch 세부사항은 workflow identity가 아니라
  typed action data다.
- patch 적용 승인은 verification command 실행 권한을 포함하지 않는다.
- 손상된 canonical state는 fail-closed 처리한다. SQLite는 재생성 가능한
  projection으로만 유지한다.
- 동시 ledger writer는 event 유실이나 chain fork 없이 직렬화한다.
- raw model output을 사용자용 최종 보고나 승인 보고로 노출하지 않는다.
- deterministic fixture와 실제 Qwen/Gemma 측정 증거를 테스트와 문서에서
  구분한다.

## 보정 순서

| 단위 | 작업 | 완료 증거 | 커밋 경계 |
| --- | --- | --- | --- |
| C1 | restart-safe patch record는 보존하면서 범용 workflow envelope와 typed action result를 추가한다. 읽기 전용 결과와 patch 결과를 별도로 처리한다. | 비패치 `run` E2E 통과, 기존 patch E2E 유지, 저장된 workflow kind 명시 | `refactor(runtime): separate workflow and patch action state` |
| C2 | 온톨로지 projection을 조회하고 source pointer를 반환하며 선택 source를 다시 읽는 runtime context provider를 도입한다. 온톨로지가 비었을 때만 제한된 filesystem fallback을 사용한다. | `run`이 ontology 선택을 기록, stale pointer 거부 또는 갱신, patch proposal이 reread source hash에 결합 | `feat(ontology): route agent context through runtime graph` |
| C3 | raw model 응답 표시를 typed 한국어 runtime 보고로 교체하고 모든 자연어 결과에 output guard를 적용한다. | read-only, 승인 대기, 차단, 종료 보고의 한국어 guard 테스트 통과, raw response 미노출 | `fix(runtime): guard all user-facing model reports` |
| C4 | patch 적용 승인과 verification-command 승인을 분리하고 두 gate를 모두 영속화한다. 각 gate에서 restart 동작을 보존한다. | patch 적용 시 verification 미실행, 두 번째 credential 필요, 두 gate의 restart/cancel 테스트 통과 | `refactor(policy): split patch and verification approvals` |
| C5 | current-state 검증을 fail-closed로 만들고 ledger mutation을 하나의 project/runtime writer lease로 직렬화한다. SQLite row 존재를 resume 권위로 사용하지 않는다. | 손상 state 거부, 동시 writer 전부 성공 및 단일 hash chain 유지, projection 재생성 후 같은 session 선택 | `fix(state): enforce canonical ledger authority` |
| C6 | approve/cancel 경쟁을 차단하고 recovery artifact를 target parent와 예상 파일명에 결합한다. | 적대적 interleaving과 변조 recovery path 테스트에서 source/state 불일치가 발생하지 않음 | `fix(patch): serialize cancellation and recovery` |
| C7 | README, HANDOFF, architecture, release notes, roadmap을 실제 검증된 capability 경계와 맞춘다. | link/marker scan 통과, 영문/한국어 문서 일치, fixture를 실제 모델 측정으로 표현하지 않음 | `docs(runtime): align v0.29 capability claims` |

## 비범위

- 기본 Qwen/Gemma 모델 승격: 출처 기반 로컬 증거가 필요한 `v0.30.0` 범위
- backend generation streaming/cancellation: `v0.31.0`
- 전체 transcript replay: `v0.32.0`
- 실행 가능한 hook, skill, TUI action, subagent, team, plugin adapter:
  `v0.33.0`부터 `v0.38.0`
- 이번 release만을 위해 canonical ontology store를 RDF/OWL로 교체하는 작업

## 출시 게이트

C1-C7 완료, targeted test 통과, 전체 locked test suite 직렬 통과, 독립
reviewer의 방향 정합성 확인 전에는 `v0.29.0`을 `main`에 merge하지 않습니다.
후속 로드맵 기능을 현재 capability로 잘못 주장하지 않는지도 함께 확인합니다.
